# SunChat / 虚拟角色对话 — 需求规格说明书（PRD）

本文档描述基于 **Python** 与 **FastAPI** 的流式角色对话服务的产品需求；**阶段二及以后**的 Agent 工具链、长期记忆 RAG 等仍以路线图形式保留，与当前仓库**已实现**的阶段一范围需区分阅读。

---

## 1. 项目概述

| 项 | 说明 |
|----|------|
| **工程名** | SunChat（仓库与 README 命名） |
| **核心定位（阶段一）** | 通过 WebSocket 与角色进行中文为主的实时对话；回复**流式**输出；角色拥有**稳定人设**，并随对话呈现**可感知的心情变化**（由心情指数与主模型演绎，而非改写人格数值）。 |
| **当前角色示例** | `system_base.txt` 与 `character_traits.json` 中的 **Yuki（由希）** 温泉职员人设（可替换为其它角色，需同步维护两份提示与结构化人设）。 |
| **设计哲学（长期）** | 「有温度的实干家」：在阶段二及以后扩展为具备工具调用与任务执行的智能体；**阶段一不实现**沙箱与 LangGraph 编排。 |

## 2. 核心功能需求

### 2.1 心情感知与角色表现（阶段一 · 已实现方向）

- **人格**：以 **Big Five（OCEAN）** 等为参考的人设描述在 **System Base** 中固定；**不因单轮聊天自动改写**底层人格数值。
- **心情**：
  - **维度（阶段一）**：采用 **单轴效价（Valence）**，经独立 **评判 LLM** 输出 `valence`、`confidence`、`label`，再映射为 **心情指数 0～100**（**50 为中性**，越高越偏积极愉快，越低越偏消极冷淡）。
  - **更新机制**：**每轮用户发言后**调用评判模型一次（输入为结构化 `character_traits.json` + 最近若干轮对话 + 当前用户句）；**不使用**跨轮 EMA（指数滑动平均）累积心情状态。
  - **表现联动**：将 **心情指数** 与 **mood_injection.txt** 中的说明一并作为 **Emotional Context** 注入主模型，由模型**结合人设自行演绎**语气与反应，**不**使用写死的三档固定语气模板。
- **短 / 长期记忆**：
  - **短期**：滑动窗口内的多轮 `user` / `assistant` 消息（可配置窗口大小；可选按会话持久化到本地文件）。
  - **长期**：向量库与自动摘要 **不在阶段一实现**（见第 5 节规划）。

### 2.2 自主 Agent 执行能力（阶段二 · 规划中）

参考 Claude Code / OpenHands：感知—决策—行动—观测闭环；任务编排、文件系统、终端、MCP 等。**当前仓库未实现**。

### 2.3 可观测与调试（阶段一）

- 可选将每轮 **心情指数、评判元数据、注入文案、助手全文** 写入本地 **JSONL 轨迹**（`data/mood_trace/`），便于分析「指数变化与回复是否一致」。

## 3. 技术架构方案（与实现对齐）

### 3.1 阶段一逻辑结构

```text
浏览器（聊天 UI + 心情指数展示）
        │  WebSocket：session / emotion / token / done / error
        ▼
FastAPI
  ├── GET /           → 静态聊天页
  ├── WebSocket /ws   → 会话与流式对话
  ├── GET /health
  ├── 短期记忆（窗口 + 可选文件持久化）
  ├── 心情：评判 LLM → valence → mood_pct（0～100）→ mood_injection
  └── 主对话 LLM：System Base + Emotional Context + History → 流式生成
```

### 3.2 关键技术选型（阶段一）

| 领域 | 选型 |
|------|------|
| 后端 | FastAPI、Uvicorn |
| 实时通道 | WebSocket（JSON 控制帧 + token 流） |
| 大模型 | OpenAI 兼容 API（如 DeepSeek），`LLMClient` 流式与非流式（评判） |
| 心情评判 | 独立 `complete_chat` 调用，结构化 JSON 输出 |
| 阶段二预留 | LangGraph、Docker 沙箱等 **未接入** |

## 4. 详细模块说明（阶段一）

### 4.1 心情链路

1. 用户发送一轮文本。
2. 服务端调用 **评判模型**（人设来自 `character_traits.json`，对话含历史 + 当前句）。
3. 解析 `valence` / `confidence` / `label`，映射为 **mood_pct**；失败时 **mood_pct = 50** 并记录，**不阻断**对话。
4. 组装 **System Base** + **mood_injection（含当前指数与概要）** + **历史**，主模型流式生成回复。

### 4.2 执行引擎（阶段二）

文件操作、终端、沙箱、自我修正等 — **需求保留，实现未纳入当前里程碑**。

## 5. 扩展性规划（Future）

- **双轴情绪**：Valence + Arousal；可与现有心情指数并存迭代。
- **多模态**：Whisper、TTS、Live2D 表情索引等。
- **长期记忆与 RAG**：向量库存储与召回。
- **IDE / 插件**：可选。

---

## 6. 非功能性需求

- **性能**：流式首包延迟可接受；评判 + 主对话两次 LLM 调用需可配置超时。
- **隐私**：密钥走环境变量；轨迹与会话数据默认在本地 `data/`（已 `.gitignore`）。
- **鲁棒性**：评判或主模型失败时，通过 WebSocket 返回 **`type: error`**，连接可保持；心情评判失败时降级为中性指数。
