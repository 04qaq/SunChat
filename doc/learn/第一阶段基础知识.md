# 一、整体技术本质

本项目的核心不是“聊天程序”，而是：

```text
一个“带状态管理的 LLM 编排系统”
```

关键点：

| 组件        | 本质    |
| --------- | ----- |
| LLM       | 无状态函数 |
| Memory    | 上下文状态 |
| 心情评判   | 每轮独立：结构化人设 + 对话 → valence → mood_pct |
| WebSocket | 实时调度层 |

## 2.1 异步编程（async / await）

### ✔ 为什么必须用异步？

你的系统涉及：

* WebSocket长连接
* HTTP请求（调用LLM）
* 流式返回

👉 如果用同步，会**阻塞整个服务**

---

## ✔ 基本语法

```python
async def func():
    await something()
```

---

## ✔ 在本项目中的体现

### WebSocket处理

```python
async def websocket_endpoint(ws: WebSocket):
    await ws.accept()
```

---

### 调用 LLM

```python
async for token in llm.stream_chat(messages):
```

---

## ✔ 核心理解

| 关键字       | 含义   |
| --------- | ---- |
| async     | 定义协程 |

| async for | 异步迭代 |

| ----- | --------- |
| 请求-响应 | 双向通信      |

# 六、心情系统（与当前仓库一致）

---

## 6.1 设计要点

本仓库**不使用**跨轮 EMA（无 \(E_t=\gamma E_{t-1}+(1-\gamma)S_{new}\)），也**不使用**三档固定「热情/冷淡」映射。

每轮流程：

```text
character_traits.json + 最近对话 + 当前用户句
        → 评判 LLM（complete_chat，JSON）
        → valence ∈ [-1,1]、confidence、label
        → mood_pct ∈ [0,100]（50 为中性；由 v_eff = valence×confidence 映射）
        → build_mood_prompt_injection + mood_injection.txt
        → 与 system_base、history 一并送入主模型流式生成
```

---

## 6.2 代码位置（便于对照）

| 文件 | 作用 |
|------|------|
| `emtion/judge.py` | 评判请求、解析 JSON、`mood_pct` |
| `emtion/state.py` | `build_mood_prompt_injection` |
| `prompts/mood_injection.txt` | 指数含义与演绎要求（非写死三档台词） |
| `prompts/character_traits.json` | 评判用人设（需与 `system_base.txt` 手动同步） |

---

## 6.3 降级

评判解析失败时：`mood_pct = 50`，对话不中断。

配置 `MOOD_USE_LLM_JUDGE=false` 时：指数恒为 50，不跑评判。

# 八、从「数值」到「Prompt」（mood_injection）

---

## 8.1 作用

```text
mood_pct（0～100）+ 可选 label → 第二条 system 文案
```

由模板说明指数含义，并要求模型**结合人设**自行演绎语气，而不是：

```text
❌ 仅写死「请热情回复」三档
✅ 给出当前指数 + 约束，由主模型与人设对齐
```

---

## 8.2 与主对话的关系

主模型同时看到 **System Base（长叙事人设）** 与 **Emotional Context（短心情条）**，需在提示词上调好权重（见产品侧讨论）。

# 十一、系统调度（WebSocket层）

---

## 11.1 核心逻辑

```python
while True:
    data = await ws.receive_json()
```

---

## ✔ 处理流程（与 ws.py 一致）

```text
用户输入
 → 评判心情（可选）→ mood_pct / label
 → 写 memory（user）
 → 构建 messages：system_base + mood_injection + history
 → 调主 LLM
 → 流式返回 token
 → 写 memory（assistant）
```
