# SunChat 开发手册

本文档描述 **SunChat** 工程的实现结构与阶段划分；**阶段一**与当前代码库一致，**阶段二～四**为规划。

---

## 1. 阶段一：核心聊天与心情注入（当前实现）

**目标：** FastAPI + WebSocket 流式对话；角色人设固定；每轮根据 **评判 LLM** 与 **心情指数（0～100）** 影响回复表现，前端展示心情指数与可选标签。

### 1.1 实时对话（FastAPI + WebSocket）

- **入口**：`run.py` 启动 Uvicorn，默认 **`HOST=0.0.0.0`**、**`PORT=8000`**（可用环境变量覆盖）。
- **页面**：`GET /` → `app/static/chat.html`。
- **连接**：`WebSocket /ws`；前端通过 `ws://主机:端口/ws` 连接，若用 `file://` 打开页面需使用查询参数 **`?ws_host=主机:端口`** 或依赖默认 `localhost:8000`。
- **流式**：`LLMClient.stream_chat` 解析 SSE，逐段 `{"type":"token","delta":...}` 推送。

### 1.2 Prompt 三层结构

| 层级 | 来源 | 说明 |
|------|------|------|
| **System Base** | `src/app/prompts/system_base.txt` | 角色叙事、语言风格、场景设定（如 Yuki / 旁白）。 |
| **Emotional Context** | `build_mood_prompt_injection` + `mood_injection.txt` | 注入 **当前心情指数 0～100** 及可选 **本轮概要**；要求模型结合人设自行演绎，**非**三档写死语气。 |
| **Chat History** | `ShortTermMemory` | 窗口内 `user` / `assistant` 消息；本轮用户句在评判之后写入记忆，再与两条 system 一并送入主模型。 |

### 1.3 心情模块（无 EMA）

- **评判**：`app/emtion/judge.py` 读取 `character_traits.json`、`judge_system.txt`，调用 `LLMClient.complete_chat`，解析 JSON：`valence` ∈ [-1,1]、`confidence`、`label`。
- **映射**：`mood_pct = round(((valence × confidence) + 1) × 50)`，限制在 0～100；**不做** \(E_t = \gamma E_{t-1} + (1-\gamma) S_{new}\) 类累积。
- **关闭评判**：配置 `MOOD_USE_LLM_JUDGE=false` 时，`mood_pct` 固定 50，且不注入评判标签行。
- **人设卡**：`character_traits.json` 供评判模型使用；与 `system_base.txt` 应保持一致，**手动同步**（无自动生成）。

### 1.4 轨迹与调试

- `MOOD_TRACE_ENABLED` / `MOOD_TRACE_DIR`：每轮追加 JSONL（心情、评判字段、注入全文、用户与助手文本）。

### 1.5 关键交付物（阶段一）

- Web 聊天页 + 心情指数展示。
- WebSocket 协议：`session`、`emotion`（`mood_pct`、`label`）、`token`、`done`、`error`。
- 配置项见 `app/config.py` 与 `.env.example`。

```text

  main.py              # 路由：/、/ws、/health
  config.py            # CHAT_WINDOW_SIZE、MOOD_*、LLM、DATA 路径等
  api/ws.py            # WebSocket 会话与对话编排
  llm/client.py        # stream_chat、complete_chat
  memory/short_term.py # 窗口与可选文件持久化
  emtion/              # 目录名历史拼写
    judge.py           # 评判与 mood_pct
    state.py           # build_mood_prompt_injection
    trace.py           # JSONL 轨迹
  prompts/
    system_base.txt
    character_traits.json
    judge_system.txt
    mood_injection.txt
  static/chat.html
run.py                 # uvicorn，HOST/PORT
```

## 避坑提示

- **两次 LLM 调用**：每轮用户消息先评判再主聊，注意超时与费用。
- **人设一致性**：修改角色时同时更新 `system_base.txt` 与 `character_traits.json`。
- **连接问题**：勿用 `file://` 直接打开 `chat.html` 而不带 `ws_host`；服务端需监听 `0.0.0.0` 以便局域网访问。

**文档版本**：与当前代码「心情指数 + 评判 LLM + 无 EMA」一致；旧版「Emotion Mapper 三档 + γ 衰减」已废止。
