# Nero 开发手册

## 1. 阶段一：灵魂构建（核心聊天与动态情绪）

**目标：** 实现一个能秒回信息、且语气会随着聊天内容变幻的虚拟形象。

### 1.1 实时对话引擎 (FastAPI + WebSocket)

- **低延迟设计**：使用流式输出（Streaming），让 Nero 的文字像人类打字一样逐字呈现。
- **Prompt 结构**：采用三层架构：
  - **System Base**：角色基本定义（性格、背景）。
  - **Emotional Context**：当前实时情绪状态。
  - **Chat History**：上下文对话缓存。

### 1.2 情绪计算模型 (Sentiment Middleware)

在消息发出前，通过中间件更新 Nero 的内心状态。

#### 情绪衰减公式

为了保证情绪的连续性，使用以下公式计算当前情绪值 $E_{t}$：

$$E_{t}=E_{t-1}\cdot\gamma+S_{new}\cdot(1-\gamma)$$

- **参数说明**：
  - $\gamma$：衰减系数（建议设为 `0.7`）。
  - $S_{new}$：当前用户话语的情感极性（范围为 $[-1,1]$，$-1$ 为极度冒犯，$1$ 为极度赞美）。

### 1.3 关键交付物

- **Emotion Mapper**：将数值映射为回复风格（例如：$>0.5$ 为“热情”，$<-0.5$ 为“毒舌/冷淡”）。
- **Web 基础界面**：一个极简聊天窗口，实时显示 Nero 的当前“心情标签”。

## 2. 阶段二：肌肉生长（Agent 执行与工具箱）

**目标：** 参考 Claude Code，让 Nero 拥有操作文件和执行代码的能力。

### 2.1 基于 LangGraph 的自主循环

设计一个简单的 **ReAct** (Reasoning and Acting) 状态机：

**节点流向**：

`Chat` (日常对话) $\rightarrow$ `Reasoning` (判断是否需要工具) $\rightarrow$ `Action` (执行工具) $\rightarrow$ `Observe` (观察结果) $\rightarrow$ `Chat`

### 2.2 工具集定义 (Tools)

- **文件操作**：`read_file`, `edit_file` (支持 diff 模式)。
- **系统交互**：`run_shell`（在沙箱中运行）。
- **搜索**：`web_search`（获取实时资讯）。

### 2.3 安全沙箱 (Docker Implementation)

所有 `run_shell` 命令必须发送至独立的 Docker 容器执行。

## 3. 阶段三：记忆增强（长期记忆与 RAG）

**目标：** 让 Nero 记得你三天前提到过的 Bug，或者你喜欢的代码风格。

### 3.1 记忆分层架构

- **短期记忆**：存储在 Redis 中的最近 20 轮对话。
- **长期记忆**：使用 Vector DB（如 ChromaDB 或 Milvus）。
- **提取逻辑**：任务结束时，由低成本模型总结关键信息并存入向量库。

### 3.2 召回机制

每轮对话前，根据用户问题检索相关的长期记忆碎片，以 `Background Information` 的形式注入 Prompt。

------

## 4. 阶段四：表现力提升（多模态与生态）

**目标：** 从文字聊天进化到“看得见、听得到”的虚拟形象。

### 4.1 视觉联动 (Live2D/3D)

- **情绪标签透传**：后端返回 JSON 需包含 `emotion_tag`（如 `laughing`, `angry`, `thinking`）。
- **动作映射**：前端触发对应的 Live2D 动作文件（`.mtn` 或 `.motion3.json`）。

### 4.2 语音进化 (TTS/ASR)

- 使用 **Edge-TTS** 提供高质量语音合成。
- **流式推流**：在文字生成时同步开启语音推流，实现边说边打字。

## 技术架构总览表

| **模块**       | **推荐技术栈**          | **优先级** |
| -------------- | ----------------------- | ---------- |
| **后端框架**   | FastAPI + LangGraph     | 高 (P0)    |
| **大模型调度** | PydanticAI / LangChain  | 高 (P0)    |
| **情绪计算**   | NLP / Custom LLM Prompt | 中 (P1)    |
| **执行环境**   | Docker SDK for Python   | 中 (P1)    |
| **存储层**     | Redis + ChromaDB        | 低 (P2)    |
| **表现层**     | React/Vue + Live2D SDK  | 低 (P2)    |

------

## 给架构师的“避坑”建议

- **聊天与任务分离**：使用异步任务 (`asyncio`)，不要让跑代码的过程阻塞聊天。
- **防止 Agent 逃逸**：严格限制容器的 CPU 和内存占用。
- **Token 成本控制**：情绪分析用极小模型（如 1B 参数量级），核心逻辑再上顶级模型。
